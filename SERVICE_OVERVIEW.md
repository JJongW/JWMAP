# JWMAP - 오늘 오디가? 서비스 개요

## 서비스 소개

**"오늘 오디가?"**는 서울 및 수도권 맛집을 큐레이션하는 웹 서비스입니다. 운영자가 직접 방문하고 검증한 장소만을 수록하여, 사용자가 빠르게 의사결정을 내리고 바로 출발할 수 있도록 돕는 것이 핵심 가치입니다.

### 서비스 정체성

- **큐레이션 기반**: 대규모 데이터베이스가 아닌, 개인이 직접 검증한 장소만 수록
- **빠른 의사결정**: 복잡한 정보 탐색 없이 핵심 정보만 제공하여 즉시 결정 가능
- **검증된 정보**: 모든 장소는 운영자의 실제 방문 경험을 바탕으로 한 정보 제공

---

## 주요 기능

### 1. 지능형 자연어 검색

Google Gemini 기반의 AI 검색 엔진으로 자연스러운 한국어 질문을 이해합니다.

**검색 예시:**
- "용산 혼밥 맛집" → 용산 지역의 혼밥 가능한 장소 검색
- "강남 라멘" → 강남 지역의 라멘 전문점 검색
- "홍대 데이트 맛집" → 홍대 지역의 데이트 장소 추천
- "야식 맛집" → 심야 영업 장소 검색
- "카공카페" → 카페에서 공부하기 좋은 장소 검색

**지원하는 검색 의도 (16가지):**
- 추천 요청, 특정 음식 검색, 카테고리 검색, 지역 검색
- 조건 검색 (혼밥 가능, 주차 가능 등), 상황 검색 (데이트, 회식 등)
- 비교, 랜덤 선택, 근처 검색, 영업중 검색, 야식 검색
- 장소 상세 질문, 유사 장소 검색, 제외 검색 등

### 2. 계층적 필터링 시스템

**지역 필터:**
- 시/도 단위: 서울, 경기, 인천, 부산 등 17개 시/도
- 세부 지역: 서울만 19개 구역 (강남, 홍대, 용산 등)
- 소분류: 동 단위까지 지원 (예: 한남동, 연남동)

**카테고리 필터:**
- 대분류 (10개): 밥, 면, 국물, 고기요리, 해산물, 간편식, 양식·퓨전, 디저트, 카페, 술안주
- 소분류 (50+): 라멘, 덮밥, 회, 이자카야 등 세부 카테고리

**특징 필터:**
- 혼밥 가능, 조용한 분위기, 웨이팅 짧음, 데이트 추천
- 단체석 있음, 주차 가능, 반려동물 동반, 예약 가능, 심야 영업 등

**이벤트 필터:**
- 특별 이벤트가 진행 중인 장소 필터링 (예: 흑백요리사 이벤트)

### 3. 지도 통합 및 네비게이션

**카카오 지도 연동:**
- 실시간 지도 표시 및 마커 시각화
- 카테고리별 커스텀 마커 아이콘 (밥집, 면집, 카페 등)
- 사용자 현재 위치 표시
- 지도와 목록 동기화 (마커 클릭 ↔ 목록 선택)

**지도 앱 딥링크:**
- 모바일: 네이버 지도/카카오맵 앱 자동 실행
- PC: 웹 지도 페이지 열기
- 앱 미설치 시 자동으로 웹 지도로 전환

### 4. 장소 상세 정보

**기본 정보:**
- 장소명, 주소, 좌표
- 평점 (1-5점), 가격대 (1-4단계)
- 카테고리, 지역, 세부 지역

**상세 정보:**
- 한줄 설명 (short_desc)
- 개인 메모 (memo) - 운영자의 상세한 방문 후기
- 방문일 (visit_date)
- 특징 태그 (features) - 혼밥 가능, 조용함 등

**이미지:**
- Cloudinary를 통한 이미지 호스팅
- 자동 이미지 압축 (2MB 미만으로 최적화)
- 갤러리 형태로 표시

**커뮤니티 리뷰:**
- 다른 사용자들의 한줄 리뷰
- 방문 유형 (첫 방문/재방문)
- 특징 태그 포함

### 5. 반응형 디자인

**모바일 (768px 미만):**
- 전체 화면 지도
- 하단 시트 형태의 장소 목록
- 풀페이지 장소 상세 뷰
- 제스처 기반 인터랙션 (스와이프, 드래그)

**데스크톱 (768px 이상):**
- 좌측 사이드바 (장소 목록/상세)
- 우측 지도 영역
- 동시에 여러 정보 확인 가능

### 6. 장소 관리 기능

**장소 추가:**
- 카카오 장소 검색 API 연동
- 수동 입력 지원
- 이미지 업로드 (자동 압축)
- 특징 태그 선택
- LLM 기반 태그 자동 추천

**장소 수정/삭제:**
- 실시간 Supabase 동기화
- 필드별 수정 가능

## 기술 스택

### Frontend
- **React 18** + **TypeScript** + **Vite**
- **Tailwind CSS** - 유틸리티 기반 스타일링
- 반응형 디자인 (모바일/데스크톱)

### Backend & Database
- **Supabase (PostgreSQL)** - 데이터베이스 및 인증
- **Vercel Serverless Functions** - API 엔드포인트

### 외부 서비스
- **Kakao Maps API** - 지도 및 장소 검색
- **Naver Maps API** - 딥링크 지원
- **Google Gemini (gemini-2.0-flash)** - 자연어 검색 파싱
- **LangChain** - LLM 프롬프트 관리
- **Cloudinary** - 이미지 호스팅 및 최적화

### 배포
- **Vercel** - 프론트엔드 및 서버리스 함수 배포

## 데이터 구조

### 장소 (Location)
- 기본 정보: 이름, 주소, 좌표, 평점, 가격대
- 분류: 지역 (province, region, sub_region), 카테고리 (category_main, category_sub)
- 특징: features (JSON), tags (배열), event_tags (배열)
- 메타데이터: 이미지 URL, 방문일, 생성일
- 외부 연동: 카카오/네이버 place_id

### 리뷰 (Review)
- 장소 ID, 사용자 표시명
- 한줄 리뷰, 방문 유형 (첫 방문/재방문)
- 특징 태그, 생성일

### 검색 로그 (Search Logs)
- 검색 쿼리, 파싱 결과 (JSON)
- 결과 개수, 성능 메트릭 (LLM 시간, DB 시간, 총 시간)

### 클릭 로그 (Click Logs)
- 검색 ID, 장소 ID, 액션 타입, 타임스탬프

## 핵심 UX 플로우

```
1. 사용자가 장소 탐색
   ├─ 지역/카테고리 필터 선택
   └─ 자연어 검색 (LLM)

2. 장소 목록 표시
   ├─ 필터링된 결과
   └─ 지도에 마커 표시

3. 지도에서 장소 선택
   └─ 장소 프리뷰 표시

4. 장소 상세 정보 확인
   ├─ 평점, 메모, 특징
   ├─ 이미지 갤러리
   └─ 커뮤니티 리뷰

5. 네비게이션 실행
   ├─ PC: 웹 지도 페이지
   └─ 모바일: 네이버/카카오 앱 딥링크
```

## 차별화 포인트

1. **검증된 큐레이션**: 개인적으로 방문하고 검증한 장소만 수록하여 신뢰도 높음
2. **자연어 검색**: 복잡한 필터 조작 없이 "용산 혼밥 맛집"처럼 자연스럽게 검색
3. **빠른 의사결정**: 핵심 정보만 제공하여 빠르게 선택 가능
4. **지도 중심 UX**: 지도와 목록이 실시간으로 동기화되어 직관적
5. **모바일 최적화**: 모바일에서도 편리하게 사용 가능한 제스처 기반 인터페이스

## 주요 사용 시나리오

1. **"오늘 점심 뭐 먹지?"**
   - 현재 위치 기반 근처 맛집 검색
   - 혼밥 가능한 곳 필터링
   - 빠르게 선택 후 바로 출발

2. **"용산에서 데이트할 곳 찾기"**
   - 지역: 용산 선택
   - 특징: 데이트 추천 필터
   - 지도에서 위치 확인 후 예약

3. **"야식 먹고 싶어"**
   - 자연어 검색: "야식 맛집"
   - 심야 영업 장소만 표시
   - 지도에서 가까운 곳 선택

4. **"카공할 카페 찾기"**
   - 자연어 검색: "카공카페"
   - 조용하고 오래 앉을 수 있는 카페 필터링
   - 위치 확인 후 바로 이동
